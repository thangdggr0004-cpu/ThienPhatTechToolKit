# ThienPhatTechToolkit Project Guidelines

This file serves as the main guideline for the `ThienPhatTechToolkit` project. It outlines general conventions, architectural patterns, and important notes for contributors.

## Project Structure
- `src/`: Contains the main source code for the Electron application.
- `assets/`: Stores static assets like images, icons, etc.
- `dist/`: Output directory for built frontend assets.
- `dist-electron/`: Output directory for Electron build.

## Development Workflow
- Follow existing coding styles and patterns.
- Ensure all new features are covered by tests where applicable.
- Use `npm` for package management.

## Sub-directory Specific Guidelines
Refer to `src/GEMINI.md` for guidelines specific to the source code.

## How to use GEMINI.md files
These `GEMINI.md` files are meant to provide context and instructions for different parts of the project. If you have specific instructions or context for a particular subdirectory, create a `GEMINI.md` file within that subdirectory. This helps keep information relevant and localized.

Remember to update these files as the project evolves.
